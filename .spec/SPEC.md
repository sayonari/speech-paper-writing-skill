# SPEC - 技術仕様・要件定義

## 目的
既存の `~/.claude/skills/paper_writing` と `~/.claude/skills/field_literature_survey` を、実コーパス（ICASSP/Interspeech論文）に基づく分析結果で**修正・アップグレード**し、AIが「AI臭くない」音声言語情報処理分野らしい英語論文を書けるようにする。

## 背景・問題意識
- 現状のAI出力には「AI臭さ」が残る：
  - `---`（em-dash多用）など人間の論文では稀な記号の使用
  - セミコロン `;` による文の接続の多用
  - 箇条書きの多用、段落の長さ・構成の不自然さ（遠目で見てわかるAI臭）
  - AI特有の語彙（delve, leverage, pivotal 等の候補を検証する）
- これらを**網羅的・徹底的に**実コーパスと照合して洗い出し、既存スキルに反映する

## 機能要件

### FR-1. コーパス構築（全年度）
- NAS（SSH経由rsync）から `_ref/papers/` へアーカイブ
  - ICASSP 2001–2026（NASにある全年度）
  - Interspeech 2007–2017, 2019–2023（NASにある全年度。2018は欠落、2020は索引のみ）
- ISCA Archive（web）から補完収集：Interspeech **2018・2020・2024・2025**（indexページ含む）
  - レート制限を設けた行儀のよいダウンロードスクリプト（scripts/download_isca.sh）で実施

### FR-1b. 3層分析設計
- **第1層（コア）2015–2022**：ChatGPT登場以前の「AI混入のない現代の人間文体」。使う表現／避ける表現リストの主根拠
- **第2層（AI混入観測）2023–2026**：AI典型語の年次頻度推移を追跡。急増語＝AI由来の疑い濃厚として避けるリストへ
- **第3層（長期トレンド）2001/2005/2010等の飛び石**：時代を超えて安定する核表現と流行語の区別
- 全PDFをテキスト化（pdftotext等）し、分析用テキストコーパスを構築（テキストは軽量なのでローカル高速ディスクに保持）

### FR-2. 統計分析（コーパス全体）
- 頻出語彙・頻出n-gram（表現・言い回し）の抽出
- 記号使用統計（em-dash、セミコロン、コロン、括弧、箇条書き記号 等）
- 文長・段落長・セクション構成の統計
- セクション別（Abstract / Introduction / Method / Experiments / Conclusion）の定型表現抽出
  - ※統計的に意味が薄ければセクション別の粒度は落とす（進めながら判断）

### FR-3. 「避けるべき表現」の特定（3手法すべて実施）
1. **AI典型表現との照合**: AIが多用しがちな語彙・記号・構文の候補リストを作成し、実コーパスでの出現頻度を測定 →「実は使われない」ものを特定
2. **AI生成論文との差分**: 実論文と同テーマでAIにセクションを書かせ、実論文との差分を分析
   - 語彙・表現レベルの差分
   - **構成レベルの差分**（段落の長さ・数、箇条書きの使用量、見出し構造など「遠目で見たAI臭」）
3. **一般学術英語との比較**: 他分野では使うが音声分野では使わない表現の抽出（余力があれば）

### FR-4. 精読（サブ分野重み付け）
- ユーザーの研究分野に近い論文（**音声対話システム・相槌（backchannel）・応答タイミング（turn-taking / response timing）**）を優先的に選定してAIが精読
- 文脈・ニュアンス・論文構成のパターンを抽出（統計では拾えない質的知見）

### FR-5. スキルのアップグレード
- 分析結果を基に以下を更新：
  - `~/.claude/skills/paper_writing/`（本体＋参照ファイルとして表現集・禁止リストを追加）
  - `~/.claude/skills/field_literature_survey/`（必要に応じて手順を改善）
- 完成品は**グローバル**（~/.claude/skills/）に設置
- 更新前に既存スキルのバックアップを本プロジェクト内に保存

## 非機能要件
- 分析スクリプトは Python、本プロジェクト内（`scripts/` 等）で管理・再実行可能にする
- PDF・大容量データは Git 管理外（.gitignore 済み）、スクリプトと分析結果（軽量なMD/CSV）は Git 管理
- Webからの収集はレート制限を守る（ISCA Archiveはオープンアクセス）
- 分析結果には根拠（頻度数値・出典論文）を残し、トレーサビリティを確保

## 技術構成
- テキスト抽出: pdftotext (poppler) / pypdf 等
- 統計分析: Python（collections, 必要なら spaCy/NLTK）
- Web収集: curl/wget or Python requests（レート制限付き）
- 成果物: Markdown（スキル本体・参照ファイル）、CSV/JSON（中間データ）

## 進め方の方針
- NASコピーの完了を待たず、**既にコピー済みの年度（ICASSP 2020, Interspeech 2020–2021 等）から分析を開始**する
- Interspeech 2024/2025 のweb収集はNASコピーと独立に並行実行
- セクション別整理の粒度、統計の閾値などは中間結果を見ながらユーザーと相談して調整

# AI生成 vs 実論文 差分分析レポート

- 実施日: 2026-07-02
- 方法: 実在論文3本と同テーマのIntroductionを、スタイル指示なしのAI（Claude Fable 5サブエージェント）に執筆させ、実論文の同セクションと比較
  - 題材1: Voice Activity Projection（turn-taking、Interspeech 2022）
  - 題材2: 相槌予測に基づく応答タイミング推定（ICASSP 2021）
  - 題材3: 多言語の相槌形態の教師なし分析（Interspeech 2022）
- 指標算出: `scripts/compare_style.py`

## 主な発見

### 1. 現世代AIは「有名なAI語」をすでに使わない
BAN語（delve, pivotal, underscore等）・セミコロン・箇条書きは、AI生成3本とも**ゼロ**だった。
→ 語彙ブラックリストだけでは現世代AIのAI臭は検出・防止できない。以下の構造的特徴が本命。

### 2. それでもem-dashは出た
AI_1の冒頭第1文に「Voice activity — knowing who is speaking and when — is ...」（3.98回/1k語）。
実論文6本分では全て0回。**挿入説明をem-dashで包む癖**は現世代でも残存。→ カンマ・括弧に置換させる。

### 3. 「人間マーカー」表現の完全欠落（最強の識別子）
実論文は It can be seen / Note that / in order to / on the other hand 等を普通に使う
（REAL_1: 6.05回/1k語、REAL_2: 1.76回/1k語）が、**AI生成3本は合計ゼロ**。
→ スキルには「禁止」だけでなく**「人間の定型表現を意図的に使う」指示**が必要。

### 4. 文が長い
AI平均文長 23.2〜28.9語（特に題材3で28.9）に対し、実論文は20.6〜24.6語。
コーパス全体の分布（平均24.5、中央値21）に対してAIは右に寄る。
→ 「中央値21語、40語超の文を連発しない」という定量指示が有効。

### 5. 修辞疑問文
AI_3はIntroduction内で修辞疑問を3回使用（What is the inventory...? Do languages share...?）。
実論文側は0回、コーパス全体でも0.23回/1k語と稀。→ 修辞疑問は原則避ける。

### 6. 段落・構成の癖（遠目のAI臭）
- AIは**均質な段落**を作る（約100〜130語×4〜5段落）。実論文は101〜252語と不均一で、必要に応じて長短が混ざる
- AIは**三点セット列挙**（(i)...(ii)...(iii)、First... Second...）を多用する傾向。3本中2本に出現
- AIのIntroは判で押したように「現象の重要性→先行研究の限界→In this paper we→貢献列挙」の同一レトリック。実論文は順序・省略に個体差が大きい
- AI生成は3本とも約500語と長さが均一。実論文は294〜1009語とテーマに応じてばらつく

## スキルへの反映方針
1. 語彙リスト（BAN/LIMIT）は「保険」として維持しつつ、**構造ルールを主軸**にする
2. 人間マーカー表現の積極使用（Note that / It can be seen that / in order to / In other words 等）
3. em-dash禁止（カンマ・括弧へ）、修辞疑問禁止
4. 文長：中央値21語を意識、40語超を連発しない
5. 段落は内容駆動で不均一に（機械的な100語×5段落を避ける）
6. 三点列挙・First/Second型の整いすぎたパラレリズムを減らす

## 効果検証（スキル適用後の3者比較、2026-07-02）

同一テーマ（VAP）で「素のAI」「speech_style_data.md を与えたAI」「実論文」のIntroを比較:

| 指標 | 素のAI | スキル適用AI | 実論文 |
|---|---|---|---|
| em-dash /1k語 | 3.98 | **0** | 0 |
| 人間マーカー /1k語 | 0 | **6.88** | 6.05 |
| 段落構成 | 102語×5（均質） | 150語×4（不均一化） | 252語×4 |
| BAN語 | 0 | 0 | 0 |

**人間マーカー率が実論文（6.05/1k）とほぼ一致（6.88/1k）**し、em-dashも消えた。スキルは機能する。

残存問題: BANフレーズ "sheds light on" が "in order to **shed some light on**" という活用形で
すり抜けた。→ 対策として語幹ベースのgrepセルフチェックを speech_style_data.md §6.5 に追加済み。

## 限界
- サンプル3本×1モデル。他モデル（GPT系）では語彙面の傾向が異なる可能性
- 実論文のIntro抽出はPDF由来のため段落境界が不完全（段落統計は参考値）
